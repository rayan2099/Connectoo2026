# Connectoo | كونكتو 📞

كونكتو (Connectoo) هو منصة عربية رائدة ومبتكرة للمكالمات الصوتية المباشرة في نفس اللحظة (RTL Real-time Voice Calls).

أهم ميزتين لمنصة كونكتو:
1. **المبدعون والمشاهير**: مكالمات فريدة وحصرية مباشرة مع فنانيك، صناع المحتوى المفضلين لديك والمؤثرين.
2. **سوق استشارة الخبراء**: تواصل صوتي فوري طارئ بدون مواعيد مسبقة وبدون شروط انتظار مع خبراء حقيقيين في القانون، الصحة، التوجيه النفسي، الميكانيكا والدعم التقني.

---

## 🚀 المميزات الرئيسية (Core Capabilities)

- **نمط المستثمر والعملاء (Clients)**: تصفح الخبراء والمشاهير حسب التصنيف والمستوى والسعر وتفاصيل المتاحين حالياً والاتصال الفوري.
- **لوحة تحكم خبير (Provider Dashboard)**: تبديل حالة الاتصال (متصل / مشغول / غير متصل) والتحقق وتلقي المكالمات الفورية وقبولها أو رفضها.
- **نظام الاتصال المدمج (Agora Voice Call Integration)**: تجربة اتصال صوتي فوري في نفس اللحظة مدعومة بقنوات Agora RTC.
- **لوحة تحكم المشرف (Admin Interface)**: قبول وتوثيق تراخيص الخبراء، إيقاف/حظر الحسابات المتعدية، ومراجعة بلاغات المستخدمين، والاطلاع على إحصائيات الاستهلاك الكلية بدقة فائقة.

---

## 🛠️ إرشادات التشغيل والإعداد (Setup & Running)

### 1. تثبيت الحزم (Installation)

لتثبيت الحزم المطلوبة والاعتمادات في بيئة التطوير، شغل الأمر التالي:
```bash
npm install
```

### 2. إعداد متغيرات البيئة (Environment Variables)

انسخ ملف `.env.example` لإنشاء ملف جديد باسم `.env` وعيّن قيم المتغيرات المناسبة:
```env
# مفتاح واجهة برمجة التطبيقات للجيميني للذكاء الاصطناعي (مستقبلي)
GEMINI_API_KEY="your_api_key_here"

# رابط الخدمة النشطة
APP_URL="http://localhost:3000"
```

### 3. تهيئة وبناء قاعدة البيانات (PostgreSQL / Supabase Setup)

لرفع قاعدة البيانات على Supabase أو أي خادم Postgres مستقل، يمكنك تنفيذ الملفات القانونية المغذية في مجلد `/db`:
1. تشغيل `db/schema.sql` لبناء الجداول والقيود والمؤشرات الفعالة.
2. تشغيل `db/seed.sql` لتغذية الجداول بحسابات المشاهير والخبراء التجريبية.

### 4. التشغيل في بيئة التطوير (Run Dev Server)

تستخدم البيئة محرك خادم Express مدمج مع Vite لدعم معالجة الواجهات البرمجية (API Routes) ودمجها بمرونة:
```bash
npm run dev
```

### 5. البناء والتشغيل للإنتاج (Production Build & Start)

لبناء الموقع كمشروع إنتاجي مدمج وتجميع خادم Express بواسطة `esbuild`:
```bash
npm run build
npm start
```
